## 7. Contingency Plans

### Cenario 1: Build Falha

**Situacao:** Build de producao falha no D-Day
**Acao:**
1. Reverter para ultimo build estavel
2. Investigar causa
3. Comunicar atraso (max 4h)
4. Retentar deploy

### Cenario 2: Bug Critico em Producao

**Situacao:** Bug critico descoberto apos deploy
**Acao:**
1. Avaliar impacto (< 15min)
2. Decidir: hotfix ou rollback
3. Se rollback: reverter em < 30min
4. Comunicar usuarios

### Cenario 3: Performance Degradada

**Situacao:** Site lento ou instavel
**Acao:**
1. Ativar CDN cache agressivo
2. Desabilitar features nao-essenciais
3. Investigar causa
4. Escalar recursos se necessario

### Cenario 4: Ataque/Seguranca

**Situacao:** Comportamento suspeito detectado
**Acao:**
1. Ativar modo manutencao
2. Investigar logs
3. Bloquear IPs suspeitos
4. Notificar stakeholders
